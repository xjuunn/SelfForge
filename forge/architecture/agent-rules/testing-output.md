# 测试、文档与输出规则

# 必跑验证

常规代码或规则变更至少执行：

```txt
cargo fmt --check
cargo test
cargo run -- validate
cargo run -- preflight
cargo run -- errors --current --open
```

纯文档变更也必须执行 `cargo run -- validate` 和中文文档检查相关验证。

# 测试覆盖

风险越高，测试越完整。共享行为、跨模块契约和用户可见流程必须覆盖：

1. 单元测试。
2. 边界测试。
3. 错误路径测试。
4. 必要时覆盖跨平台行为。

# 文档规则

1. Markdown 必须中文可读。
2. 禁止英文占位文档。
3. 禁止乱码文档。
4. 禁止 Emoji。
5. `Agents.md` 只保留入口规则和按需阅读索引。
6. 详细设计和长规则写入 `forge/architecture/`。

# 输出规则

每轮最终说明应简洁包含：

1. 当前目标。
2. 修改内容。
3. 验证结果。
4. PR 或提交信息。
5. 未完成事项或阻断原因。
6. 任务进度。
7. 未完成任务列表。

不要把大段命令输出原样贴给用户，只汇总关键结论。

# 进度格式

最终回复结尾必须追加：

```md
任务进度：
- 已完成：...
- 进行中：...
- 未完成：...

未完成任务列表：
- ...
```

若没有未完成任务，写 `无`。
