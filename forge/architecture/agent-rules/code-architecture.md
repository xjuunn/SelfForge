# 代码架构规则

# 大文件拆分

1. 文件过大或新增逻辑明显独立时，必须优先拆分到清晰模块。
2. CLI 入口只保留命令分发、参数解析调用和结果展示，不承载业务流程。
3. 新模块必须按领域命名，公开小接口，隐藏内部辅助函数。
4. 拆分必须保持默认行为兼容，并补充定向测试。
5. 禁止为了快速完成任务继续向大文件堆叠无关逻辑。

# 模块边界

1. 应用用例进入 `src/app/`。
2. CLI 展示辅助进入 `src/cli/`。
3. Runtime、Supervisor、State、Version 保持各自职责。
4. 结构化数据优先使用类型和解析器，禁止脆弱字符串拼接替代领域模型。
