# v0.1 版本记录

# 记录规则

- 本文件集中记录 `v0.1.x` 的 patch 更新，避免为每次小版本生成独立版本文件。
- 只有 minor 或 major 变化时，才创建新的版本系列文件。
- 旧的 `v0.1.0.md`、`v0.1.1.md`、`v0.1.2.md` 已合并到本文件，确认后可删除。

## v0.1.0

# 版本变化

- 建立语义化版本规则下的稳定基线。
- 将当前稳定版本从旧代际命名 `v1` 映射为 `v0.1.0`。
- 在 `state/state.json` 中记录 `legacy_versions`，保留 `v1/v2` 历史线索。

# 新增功能

- 作为 `v0.1.1` patch 候选版本的父版本。

# 修复内容

- 无。

## v0.1.1

# 版本变化

- 引入 `vMAJOR.MINOR.PATCH` 版本规则。
- 默认进化策略由代际递增改为 patch 递增，例如 `v0.1.0 -> v0.1.1`。
- `state/state.json` 增加 `version_scheme` 与 `legacy_versions`。
- CLI `evolve` 支持 `--patch`、`--minor`、`--major`，默认使用 `--patch`。
- 生成 `v0.1.1` 候选版本，当前稳定版本保持为 `v0.1.0`。

# 新增功能

- 语义化版本解析、校验和递增。
- `evolve` 输出 commit version。
- 候选文档存在时保留，避免覆盖先写入的任务计划。

# 修复内容

- 修复格式化差异。
- 修复候选生成覆盖已有任务文档的问题。

## v0.1.4

# 版本变化

- 新增最小候选运行闭环。
- 使用 `cycle` 将 `v0.1.3` 从候选提升为当前稳定版本。
- 生成 `v0.1.4` 候选版本，继续保持 patch 级演进。
- `state/state.json` 保持 `current_version=v0.1.3`，并记录 `candidate_version=v0.1.4`。

# 新增功能

- `cycle` CLI 命令：验证当前版本与候选版本，成功则提升，失败则回滚。
- `rollback [reason]` CLI 命令：显式清空候选状态并保留当前稳定版本。
- `RollbackReport`、`CycleResult`、`CycleReport` 运行报告结构。
- 候选失败回滚测试，验证当前稳定版本不会被污染。

# 修复内容

- 修复闭环命令缺失导致候选切换仍需人工拆分执行的问题。

## v0.1.5

# 版本变化

- 新增 Runtime 工作区受控执行能力。
- 使用 `cycle` 将 `v0.1.4` 提升为当前稳定版本。
- 生成 `v0.1.5` 候选版本，继续保持 patch 级演进。
- `state/state.json` 保持 `current_version=v0.1.4`，并记录 `candidate_version=v0.1.5`。

# 新增功能

- `run` CLI 命令：在指定版本工作区内直接启动命令。
- `ExecutionReport`：记录版本、工作区、命令、参数、退出码、标准输出、标准错误和超时状态。
- `ExecutionError`：区分空命令、工作区逃逸、启动失败和等待失败。
- Runtime 超时控制：超时后终止子进程并返回超时报告。
- Runtime 工作区边界校验：拒绝逃逸出 `workspaces` 根目录的版本路径。
- 新增 4 个 Runtime 执行测试，覆盖成功、超时、空命令和路径逃逸。

# 修复内容

- 修复 `run` CLI 初版错误传播导致的编译失败。
- 修复严格检查中出现的格式化差异和 warning。
