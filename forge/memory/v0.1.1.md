# 目标

重新设计 SelfForge 的版本号规则：统一使用 `vMAJOR.MINOR.PATCH`，默认只递增 patch，例如 `v0.1.0 -> v0.1.1`；minor/major 必须显式选择，大版本升级必须克制。提交信息也必须显示本轮版本号。

# 优化建议

1. 下一步实现候选进程启动，但仍不要自动切换稳定版本。
2. 在切换前加入版本规则校验，禁止无明确理由的 major 升级。
3. 将 commit message 生成建议纳入 evolve 报告。
4. 后续清理旧模板编码问题，但应作为单独小任务处理。

# 可复用经验

1. 版本升级默认 patch，minor/major 必须显式表达意图。
2. 版本号属于审计数据，必须出现在 state、forge 文档和 commit 信息中。
3. 候选文档一旦存在，自动流程不能覆盖人工或前序流程写入的计划。
4. 旧版本记录不应被批量删除或重命名，应该通过状态中的 legacy_versions 保留可追溯关系。
