将所有 Markdown 文档修正为中文可读内容，把中文文档规则写入 `Agents.md`，并继续推进系统能力：新增中文文档审计和候选版本提升能力，使文档规范成为运行时验证的一部分。

# 计划（Plan）

1. 读取历史记忆、状态和现有文档。
2. 写入 `forge/tasks/v0.1.2.md`。
3. 实现候选提升状态迁移。
4. 实现中文文档审计模块。
5. 将文档审计接入版本验证。
6. 修复所有不合规 Markdown 文档。
7. 将文档规则和版本规则写入 `Agents.md`。
8. 执行格式化、编译、测试、验证。
9. 提升 `v0.1.1` 为当前版本，并生成 `v0.1.2` 候选。
10. 写入错误、版本和记忆记录。

# 执行过程

1. 读取当前状态，确认 `v0.1.0` 为稳定版本、`v0.1.1` 为候选版本。
2. 修复 `Agents.md` 的乱码内容，写入中文文档规则、语义化版本规则和提交规范。
3. 将 `runtime`、`supervisor`、`workspaces`、`forge` 下的 Markdown 文档改为中文可读内容。
4. 新增 `src/documentation.rs`，递归检查 Markdown 文档是否包含中文、是否含明显乱码标记。
5. 更新 `src/layout.rs`，将中文文档审计接入 `validate`。
6. 更新 `src/evolution.rs`、`src/supervisor.rs`、`src/main.rs`，新增候选提升能力和 `promote` 命令。

# 可复用经验

1. 文档规范必须进入验证流程，否则乱码和英文占位会反复出现。
2. 测试夹具也必须遵守文档规则，否则测试会与真实规则不一致。
3. 候选提升只更新状态，不应隐式生成新候选版本。
4. 先提升已验证候选，再生成下一 patch 候选，可以保持版本链清晰。
