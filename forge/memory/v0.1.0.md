# 版本信息
- 版本号：v0.1.0
- 时间：2026-04-29
- 父版本：v1/v2（历史代际命名）

# 目标

建立语义化版本规则下的稳定基线，将旧的 `v1/v2` 代际命名收敛到 `vMAJOR.MINOR.PATCH`，并以 `v0.1.0` 表示当前稳定版本。

# 计划（Plan）

1. 保留历史 `v1/v2` 归档，避免大范围重命名和迁移风险。
2. 创建 `v0.1.0` workspace 与 forge 文档作为新版本规则基线。
3. 更新 state/state.json，使 current_version 指向 `v0.1.0`。
4. 在状态中记录 legacy_versions，保留旧版本线索。
5. 继续由后续 `v0.1.1` 候选版本推进功能。

# 执行过程

通过 `cargo run -- init` 创建 `v0.1.0` 所需目录与文档，并手动迁移 state/state.json 的当前版本字段。

# 代码变更

- CURRENT_VERSION 更新为 `v0.1.0`。
- 新增语义化版本解析与递增规则。

# 测试结果

由 `v0.1.1` 本轮验证统一记录。

# 错误总结

无独立错误。

# 评估

`v0.1.0` 是语义化版本规则下的稳定锚点，不代表大版本升级，只是命名规则迁移后的基线。

# 优化建议

后续默认使用 patch 递增，只有明确的兼容性破坏或架构阶段变化才考虑 major。

# 可复用经验

历史归档可以保留，状态文件负责声明当前有效版本规则，避免通过批量重命名破坏审计链。
