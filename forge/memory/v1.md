# 版本信息
- 版本号：v1
- 时间：2026-04-29
- 父版本：无

# 目标

建立 SelfForge 的最基础架构。

# 计划（Plan）

1. 建立标准目录结构。
2. 实现 Rust CLI、Runtime 验证层与 Supervisor 编排层。
3. 写入持久化状态与 forge 文档。
4. 编写并执行测试。
5. 记录版本与提交。

# 执行过程

1. 读取当前仓库，确认没有历史 forge 记忆，当前仅有最小 Rust crate。
2. 写入 v1 任务文档，明确输入、输出与可验证计划。
3. 实现 layout、runtime、supervisor 三个 Rust 模块。
4. 将 CLI 拆为 init、validate、status 三个命令。
5. 执行 `cargo test` 验证单元、边界、错误场景。
6. 执行 `cargo run -- init` 生成 v1 目录、状态与归档文件。
7. 执行 `cargo run -- validate` 验证基础架构完整性。

# 代码变更

- 新增 `src/lib.rs` 作为库入口。
- 新增 `src/layout.rs` 负责标准目录、种子文件、状态文件与验证。
- 新增 `src/runtime.rs` 作为受控 Runtime 验证层。
- 新增 `src/supervisor.rs` 作为初始化与验证编排层。
- 更新 `src/main.rs`，提供 `init`、`validate`、`status` CLI。
- 更新 `Cargo.toml`，规范库与二进制 crate 名称。
- 新增 runtime、supervisor、workspaces、forge、state 基础目录与文档。

# 测试结果

- `cargo test`：通过，3 passed，0 failed。
- `cargo run -- init`：通过，生成 v1 基础架构。
- `cargo run -- validate`：通过，检查 19 个必需路径。

# 错误总结

- 未发现阻塞错误。
- `rg --files` 在当前 Windows 环境中被拒绝执行，已记录至 `forge/errors/v1/error-001.md`，并改用 PowerShell 原生命令读取文件；该问题不影响 SelfForge v1 架构。

# 评估

v1 已具备最基础的可运行、可验证、可解释、可审计骨架。当前版本尚未实现真实沙箱执行、资源限制、跨版本并行验证和回滚，只建立了后续演进所需的稳定边界。

# 优化建议

1. 下一版本优先补齐 workspace 级沙箱执行接口。
2. 引入结构化状态读写与状态迁移校验。
3. 增加 Supervisor 对 v1 到 v2 的启动、验证、切换、回滚流程。
4. 将错误记录从 README 升级为自动生成的 error-XXX.md。

# 可复用经验

1. 基础架构优先保持小而可测，避免一开始引入复杂自进化逻辑。
2. 所有生成文件使用幂等写入，避免重复初始化破坏人工记录。
3. Runtime 先承担验证边界，后续再扩展为沙箱执行边界。
