# 版本信息
- 版本号：v0.1.5
- 时间：2026-04-29
- 父版本：v0.1.4

# 目标

完成 SelfForge 最小可运行版本的执行环节：Runtime 可以在指定版本工作区内直接运行命令，限制工作目录到 workspace，记录退出码、标准输出、标准错误和超时状态，并通过严格测试验证正常路径、边界路径和错误路径。

# 计划（Plan）

1. 读取最近版本记忆和当前状态。
2. 将 `v0.1.4` 候选通过 `cycle` 提升为当前稳定版本。
3. 在 Runtime 中新增执行报告与执行错误类型。
4. 实现工作区路径边界校验，拒绝逃逸出 `workspaces` 的版本路径。
5. 实现无 shell 的直接命令启动，并固定当前目录为版本工作区。
6. 实现超时检测，超时后终止子进程并返回超时报告。
7. 在 Supervisor 和 CLI 中暴露 `run` 命令。
8. 增加严格测试：正常执行、即时超时、空命令、路径逃逸。
9. 执行格式化、编译、测试、Runtime 验证和真实 CLI 执行验证。
10. 生成 `v0.1.5` 候选版本并补齐归档。

# 执行过程

1. 执行 `cargo run -- cycle`，将 `v0.1.4` 提升为当前稳定版本。
2. 在 `src/runtime.rs` 中新增 `ExecutionReport` 和 `ExecutionError`。
3. 在 Runtime 中实现 `execute_in_workspace`，使用 `Command` 直接启动程序，不做隐式 shell 包装。
4. 使用 canonical path 校验工作区必须位于 `workspaces` 根目录之下。
5. 使用 `try_wait` 与超时轮询保护子进程，超时后调用 `kill` 并收集输出。
6. 在 `src/supervisor.rs` 增加执行代理入口。
7. 在 `src/main.rs` 增加 `run [--current|--candidate|--version VERSION] [--timeout-ms N] -- PROGRAM [ARGS...]`。
8. 增加 4 个 Runtime 执行相关测试，将测试总数提高到 18 个。
9. 首轮测试发现格式化差异、CLI `?` 使用错误和一个 warning，全部修复后重新验证通过。
10. 执行真实命令 `run --current --timeout-ms 5000 -- selfforge.exe help`，确认在 `workspaces/v0.1.4` 内运行成功。
11. 执行 `cargo run -- evolve --patch "实现最小 Runtime 工作区执行"`，生成 `v0.1.5` 候选版本。

# 代码变更

- `src/runtime.rs`：新增工作区受控执行、超时控制、执行报告和执行错误。
- `src/supervisor.rs`：新增 `execute_in_workspace` 入口。
- `src/main.rs`：新增 `run` 命令和参数解析。
- `src/lib.rs`：导出执行类型，更新当前版本常量，新增 Runtime 执行测试。
- `Cargo.toml`：包版本同步到 `0.1.5`。
- `Agents.md`：写入最小 Runtime 执行规则。

# 测试结果

- `cargo fmt --check`：首次失败，执行 `cargo fmt` 后通过。
- `cargo check`：首次失败，修复 CLI 错误后通过；清理 warning 后再次通过。
- `cargo test`：通过，18 个测试全部通过。
- `cargo run -- validate`：通过，SelfForge `v0.1.4` 校验 19 个必需路径。
- `cargo run -- run --current --timeout-ms 5000 -- <selfforge.exe> help`：通过，退出码为 0，未超时。
- `cargo run -- evolve --patch "实现最小 Runtime 工作区执行"`：通过，生成 `v0.1.5` 候选版本。

# 错误总结

- `forge/errors/v0.1.5/error-001.md`：格式化检查失败，已执行 `cargo fmt` 修复。
- `forge/errors/v0.1.5/error-002.md`：CLI `run` 分支在 `main` 中直接使用 `?`，已改为显式错误退出。
- `forge/errors/v0.1.5/error-003.md`：修复后仍有 unreachable warning，已去掉多余 `return` 并重新验证。

# 评估

`v0.1.5` 让 SelfForge 具备更完整的最小可运行能力：不仅能生成、验证、提升和回滚候选版本，还能在受控工作区内执行明确命令并获得可审计报告。当前执行隔离仍是最小级别，已具备工作目录限制、路径边界检查、无 shell 启动和超时终止；更强的 CPU、内存和系统调用隔离应在后续小版本中继续实现。

# 优化建议

1. 下一步将执行报告持久化到 `forge`，记录每次运行的命令、耗时和输出摘要。
2. 再下一步将 `cycle` 与 Runtime 执行报告关联起来，使候选验证过程更可审计。
3. 后续引入平台相关资源限制，分别处理 Windows、macOS、Linux。

# 可复用经验

1. 最小可运行版本应优先补齐可执行闭环，而不是一次性追求完整沙箱。
2. `run` 不应隐式调用 shell，命令和参数必须明确分离。
3. 严格测试不仅要覆盖成功路径，还要覆盖超时、空命令和路径逃逸。
