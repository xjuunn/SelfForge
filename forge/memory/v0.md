# v0 记忆记录

# 记录规则

- 本文件集中记录 v0 大版本下的小版本记忆。
- 每次 patch 或 minor 更新只追加一个版本小节，禁止创建新的小版本记忆文件。

## v0.1.8

# 版本信息
- 版本号：v0.1.8
- 时间：unix:1777479846
- 父版本：v0.1.7

# 目标

设计并约束 workspace 内部目录结构

# 计划（Plan）

2. 验证当前稳定版本仍可运行。
3. 设计 workspace 内部固定分区。
4. 在 layout 中创建并验证 workspace 骨架。
5. 增加防止 workspace 根目录杂乱的测试。
6. 更新中文规则和工作区 README。
7. 生成并提升候选版本。

# 执行过程

已为 `workspaces/v0` 创建固定内部骨架，并通过 layout 验证禁止未登记文件直接出现在 workspace 根目录。候选版本已通过 cycle 提升为当前版本。

# 代码变更

1. `layout` 新增 workspace 固定目录：`source/`、`tests/`、`sandbox/`、`artifacts/`、`logs/`。
2. `layout` 新增 workspace 根目录白名单校验，拒绝未规划的根目录文件或目录。
3. `workspace` 新增 `.gitignore` 和各分区中文 README。
4. 测试新增 workspace 骨架断言和根目录杂散文件错误测试。
5. `Agents.md` 与架构文档补充 workspace 内部分区规则。

# 测试结果

1. `cargo fmt --check`：通过。
2. `cargo check`：通过。
3. `cargo test`：通过，23 个测试全部成功。
4. `cargo run -- init`：通过，创建 workspace 内部骨架。
5. `cargo run -- advance "设计并约束 workspace 内部目录结构"`：通过，生成候选版本 `v0.1.8`。
6. `cargo run -- cycle`：通过，已将 `v0.1.8` 提升为当前版本。
7. `cargo run -- validate`：通过，检查 29 个路径。

# 错误总结

本轮未出现新增功能错误。新增错误测试验证了 workspace 根目录杂散文件会被拒绝。

# 评估

版本 `v0.1.8` 已完成最小闭环提升。workspace 根目录现在只有固定入口，后续文件必须进入明确职责目录继续分层，能降低大版本聚合后文件过多的风险。

# 优化建议

后续可继续为 `sandbox` 增加按 run id 创建执行目录的运行时 API，并把资源限制记录写入 `forge/errors` 或 `forge/memory`。

# 可复用经验

major workspace 必须配合内部职责分区，否则虽然减少了小版本目录，却会把复杂度转移到单个 workspace 根目录。
