# v0 错误记录

# 记录规则

- 本文件集中记录 v0 大版本下的小版本错误。
- 每个错误仍需包含错误信息、出现阶段、原因分析、解决方案、是否已解决。
- 小版本更新只追加错误小节，禁止创建新的小版本错误目录。

## v0.1.7

### error-001

# 错误信息

`cargo check` 首次执行时报出 `E0308` 类型不匹配：`EvolutionError::Io` 需要 `PathBuf`，但 `append_record` 中传入了 `&Path`。

# 出现阶段

代码编译验证。

# 原因分析

`append_record` 接收 `&Path`，读取文件失败时错误记录需要拥有路径所有权，旧写法使用 `path.clone()` 只能得到 `&Path`。

# 解决方案

将错误路径改为 `path.to_path_buf()`，并保持 `fs::read_to_string(path)` 使用借用路径读取。

# 是否已解决

是。后续 `cargo check` 和 `cargo test` 均已通过。

### error-002

# 错误信息

`cargo fmt --check` 首次执行发现 `src/layout.rs` 和 `src/lib.rs` 存在格式差异。

# 出现阶段

格式验证。

# 原因分析

手工调整测试和布局代码后，部分链式路径调用未按 rustfmt 规则收敛。

# 解决方案

执行 `cargo fmt` 统一格式化。

# 是否已解决

是。已重新执行 `cargo fmt --check` 并通过。
