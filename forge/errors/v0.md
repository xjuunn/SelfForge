# 记录规则

- 本文件集中记录 v0 大版本下的小版本错误。
- 每个错误仍需包含错误信息、出现阶段、原因分析、解决方案、是否已解决。

## v0.1.7

### error-001

# 错误信息

`cargo check` 首次执行时报出 `E0308` 类型不匹配：`EvolutionError::Io` 需要 `PathBuf`，但 `append_record` 中传入了 `&Path`。

# 出现阶段

代码编译验证。

# 原因分析

`append_record` 接收 `&Path`，读取文件失败时错误记录需要拥有路径所有权，旧写法使用 `path.clone()` 只能得到 `&Path`。

# 解决方案

将错误路径改为 `path.to_path_buf()`，并保持 `fs::read_to_string(path)` 使用借用路径读取。

# 是否已解决

是。后续 `cargo check` 和 `cargo test` 均已通过。

### error-002

# 错误信息

# 出现阶段

# 原因分析

手工调整测试和布局代码后，部分链式路径调用未按 rustfmt 规则收敛。

# 解决方案

执行 `cargo fmt` 统一格式化。

# 是否已解决

是。已重新执行 `cargo fmt --check` 并通过。

## v0.1.8

# 错误信息

本轮实现阶段未出现新增错误。新增错误测试用于确认 workspace 根目录出现未规划文件时会被验证流程拒绝。

# 出现阶段

布局验证设计。

# 原因分析

如果 major workspace 长期复用但根目录不受约束，生成文件容易直接堆积在 `workspaces/v0`，导致查找困难和边界混乱。

# 解决方案

在 `layout` 中新增 workspace 根目录白名单校验，并要求文件进入固定职责目录继续分层。

# 是否已解决

是。`validation_rejects_unplanned_workspace_root_entries` 已覆盖该错误路径。

## v0.1.9

# 错误信息

本轮实现阶段未出现新增错误。新增错误测试用于确认 Markdown 文档出现 Emoji 时会被验证流程拒绝。

# 出现阶段

文档规范验证。

# 原因分析

如果只在规则中声明禁止 Emoji，后续文档仍可能无意引入 Emoji，导致风格和审计格式不稳定。

# 解决方案

# 是否已解决

是。`cargo test` 已覆盖该错误路径。

## v0.1.10

# 错误信息

首次测试时，`runtime_executes_command_inside_version_workspace` 使用未规范化的 temp 路径断言 `run_dir`，而 Runtime 返回的是 canonical workspace 路径，导致断言失败。

# 出现阶段

单元测试验证。

# 原因分析

Runtime 为了防止工作区逃逸，会对 workspace 执行 canonicalize。测试直接使用 temp 根目录拼接路径，没有与 Runtime 返回路径保持同一规范。

# 解决方案

将测试改为验证 `run_dir` 位于 `report.workspace` 下，并检查父目录为 `runs`，不再混用未规范化路径。

# 是否已解决

是。修复后 `cargo test` 通过，24 个测试全部成功。
