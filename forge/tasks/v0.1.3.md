# 任务来源

用户要求继续完善 SelfForge，并调整版本记录策略：每次小版本更新写入同一个版本系列文件，避免生成过多版本文件，同时清理旧版无效版本文件。

# 任务描述

将 `forge/versions` 的记录方式从“每个 patch 一个文件”调整为“每个 minor 系列一个文件”。本轮目标版本为 `v0.1.3`，属于克制的 patch 更新，不升级 minor 或 major。

# 输入

- 最近记忆：`forge/memory/v0.1.0.md`、`forge/memory/v0.1.1.md`、`forge/memory/v0.1.2.md`
- 当前状态：`state/state.json`
- 当前版本记录：`forge/versions/v0.1.0.md`、`forge/versions/v0.1.1.md`、`forge/versions/v0.1.2.md`、`forge/versions/v1.md`、`forge/versions/v2.md`
- 用户目标：小版本更新合并到单个文件，并删除旧版没用的版本文件

# 输出

- `forge/versions/v0.1.md`
- `forge/versions/legacy.md`
- 更新后的版本记录生成逻辑
- 更新后的中文代理规则
- `v0.1.3` 候选版本文档与状态

# 计划（Plan）

1. 读取历史记忆、状态和现有版本记录。
2. 新增语义化版本系列文件名计算能力。
3. 将 Runtime 布局校验和候选文档生成改为使用 `forge/versions/vMAJOR.MINOR.md`。
4. 将 patch 候选版本记录追加到系列文件，避免创建 `vMAJOR.MINOR.PATCH.md`。
5. 补充单元测试、边界测试和错误测试。
6. 将 `Agents.md` 中的版本归档规则改为中文的小版本系列单文件规则。
7. 合并旧版本记录到 `v0.1.md` 与 `legacy.md`。
8. 经确认后删除旧的 patch 级版本文件和旧式代际版本文件。
9. 执行格式化、编译、测试和运行时验证。
10. 提升已验证候选并生成 `v0.1.3` 候选版本。
