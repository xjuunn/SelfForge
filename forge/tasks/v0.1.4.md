# 任务来源

用户要求继续完成系统功能，更进一步实现 SelfForge 的最小可运行版本。

# 任务描述

在不升级 minor 或 major 的前提下，为 SelfForge 增加最小运行闭环：候选版本存在时，系统可以通过一个命令验证当前稳定版本与候选版本；验证成功则提升候选版本，验证失败则回滚到当前稳定版本并持久化状态。

# 输入

- 当前稳定版本：v0.1.2
- 当前候选版本：v0.1.3
- 最近记忆：`forge/memory/v0.1.1.md`、`forge/memory/v0.1.2.md`、`forge/memory/v0.1.3.md`
- 状态文件：`state/state.json`

# 输出

- `cycle` CLI 命令
- `rollback` CLI 命令
- 候选运行闭环报告结构
- 回滚状态持久化能力
- 对应单元测试、边界测试和错误测试

# 计划（Plan）

1. 读取当前状态和最近版本记忆。
2. 在进化引擎中新增候选回滚能力。
3. 在进化引擎中新增候选运行闭环：验证当前版本、验证候选版本、成功提升、失败回滚。
4. 在 Supervisor 中暴露闭环运行和回滚入口。
5. 在 CLI 中新增 `cycle` 与 `rollback` 命令。
6. 增加测试覆盖成功提升、缺失候选错误、失败回滚。
7. 执行格式化、编译、测试和 Runtime 验证。
8. 使用 `cycle` 提升 `v0.1.3`，再生成 `v0.1.4` 候选。
9. 补齐记忆、版本记录和错误记录。
