# 任务来源

用户要求重新设计 SelfForge 版本号规则，改为 `v0.1.1` 这样的语义化命名；版本号必须显示在提交信息中；非必要不升级大版本号，多数修改只升级小版本号，并继续完成系统。

# 任务描述

建立克制的语义化版本规则，并将自我进化流程从旧的 `v1/v2` 递增规则迁移到 `vMAJOR.MINOR.PATCH`。本轮以 `v0.1.0` 作为稳定基线，以 `v0.1.1` 作为下一候选版本，默认只递增 patch。

# 输入

- 当前状态：state/state.json 原记录 `v1 -> v2`
- 历史记录：forge/memory/v1.md、forge/memory/v2.md
- 用户新规则：使用 `v0.1.1` 格式，提交信息包含版本号，大版本升级必须克制

# 输出

- 版本解析与递增模块
- 默认 patch 递增的 evolve 流程
- `v0.1.0` 稳定基线与 `v0.1.1` 候选版本
- 更新状态文件与 forge 文档
- 单元测试、边界测试、错误测试

# 计划（Plan）

1. 读取历史记忆和当前状态。
2. 新增语义化版本规则：`vMAJOR.MINOR.PATCH`。
3. 默认使用 patch 递增，minor/major 只作为显式策略保留。
4. 更新 evolve，使其生成 `v0.1.1` 这样的候选版本。
5. 更新 CURRENT_VERSION 为 `v0.1.0`，创建稳定基线文档。
6. 将 state/state.json 迁移为 `current_version=v0.1.0`，候选为 `v0.1.1`。
7. 补齐测试，验证 patch、minor、major、非法版本错误和候选任务文档不被覆盖。
8. 执行格式化、编译、测试、验证与真实 evolve。
9. 写入 memory、versions、errors，并用包含 `v0.1.1` 的中文 commit 信息提交。
