# 任务来源

用户要求重新审视 SelfForge 的目录结构、架构分层和可扩展性，并继续完成最小可运行闭环。重点要求：结构专业、扩展方便、使用简单、不容易出错，便于 AI 后续在清晰边界内开发。

# 任务描述

在不进行大规模重构的前提下，为 SelfForge 引入清晰的应用层入口和项目结构规则。新增 `src/app/` 作为应用用例层，提供一个高级 `advance` 闭环命令：当存在候选版本时先验证并提升或回滚，成功提升后再生成下一候选；当不存在候选版本时直接生成下一候选。

# 输入

- 当前稳定版本：v0.1.5
- 最近记忆：`forge/memory/v0.1.3.md`、`forge/memory/v0.1.4.md`、`forge/memory/v0.1.5.md`
- 状态文件：`state/state.json`
- 用户目标：更合理的项目结构、顶级架构思维、极致可维护性、继续完成最小可运行闭环

# 输出

- `src/app/` 应用层目录
- `SelfForgeApp` 应用服务
- `advance` CLI 命令
- 项目结构说明文档
- Git 仓库基础结构规则
- 单元测试、边界测试、错误测试

# 计划（Plan）

1. 提升已验证的 `v0.1.5` 候选版本为当前稳定版本。
2. 写入本任务文档和项目结构架构文档。
3. 新增 `src/app/`，将高层闭环用例放入应用层。
4. 实现 `SelfForgeApp::advance`：候选存在时先运行 `cycle`，成功后生成下一候选；失败回滚后停止。
5. 在 CLI 中新增 `advance [goal]`，作为最小可运行闭环的简单入口。
6. 更新 `Agents.md`，明确后续功能优先进入应用层用例，再连接底层模块。
7. 增加测试覆盖无候选生成、候选成功提升后继续生成、候选失败回滚后停止。
8. 执行格式化、编译、测试、Runtime 验证和真实 `advance` 命令。
9. 生成 `v0.1.6` 候选版本并补齐记忆、版本和错误记录。
