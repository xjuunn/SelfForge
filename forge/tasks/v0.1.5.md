# 任务来源

用户要求继续完成系统功能，完成最小可运行版本，并要求每一步都通过严格测试。

# 任务描述

在 `v0.1.5` 中补齐 SelfForge 最小可运行版本的执行环节：Runtime 必须能够在指定版本工作区内直接启动受控命令，记录退出码、标准输出、标准错误和超时状态；命令必须以工作区为当前目录运行，并拒绝逃逸出 `workspaces` 根目录的版本路径。

# 输入

- 当前稳定版本：v0.1.4
- 最近记忆：`forge/memory/v0.1.2.md`、`forge/memory/v0.1.3.md`、`forge/memory/v0.1.4.md`
- 状态文件：`state/state.json`
- 用户目标：完成最小可运行版本，严格测试

# 输出

- Runtime 工作区命令执行接口
- `run` CLI 命令
- 执行报告结构
- 超时控制
- 工作区边界校验
- 单元测试、边界测试、错误测试

# 计划（Plan）

1. 读取历史记忆和当前状态。
2. 设计最小 Runtime 执行报告与错误类型。
3. 实现工作区路径边界校验，确保执行目录位于 `workspaces` 下。
4. 实现无 shell 的直接命令启动，并将当前目录固定为目标版本工作区。
5. 实现超时检测，超时后终止子进程并返回超时报告。
6. 在 Supervisor 和 CLI 中暴露 `run` 命令。
7. 增加测试：正常执行、即时超时、非法版本路径、缺失命令。
8. 每个实现阶段后执行格式化、编译、测试和 Runtime 验证。
9. 生成 `v0.1.5` 候选并补齐记忆、版本和错误记录。
