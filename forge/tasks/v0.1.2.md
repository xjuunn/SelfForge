# 任务来源

用户要求所有文档都必须使用中文，将不合规范的文档改为正确内容，继续完成更多系统功能，并将规则写入 `Agents.md`。

# 任务描述

在 `v0.1.2` 中完成文档规范化与验证能力：修复现有英文和乱码文档；将中文文档规则写入 `Agents.md`；新增运行时文档审计，确保未来 `validate` 能发现非中文或乱码 Markdown 文档。同时补齐候选提升能力，使已验证候选版本可以进入当前版本状态，为继续生成下一候选版本铺路。

# 输入

- 当前稳定版本：`v0.1.0`
- 当前候选版本：`v0.1.1`
- 历史文档：`forge`、`workspaces`、`runtime`、`supervisor`、`Agents.md`
- 用户规则：所有文档必须使用中文，提交信息必须包含语义化版本号

# 输出

- 全部 Markdown 文档改为中文可读内容
- `Agents.md` 写入中文文档规则与语义化版本规则
- 中文文档审计模块
- 候选提升命令
- `v0.1.1` 提升为当前版本，生成 `v0.1.2` 候选版本
- 单元测试、边界测试、错误测试

# 计划（Plan）

1. 读取历史记忆、状态和现有 Markdown 文档。
2. 实现候选提升状态迁移，支持将 `candidate_version` 提升为 `current_version`。
3. 实现中文文档审计，拒绝无中文内容或含明显乱码标记的 Markdown 文档。
4. 将文档审计接入验证流程。
5. 修复所有不合规文档，并将规则写入 `Agents.md`。
6. 执行格式化、编译、测试和验证。
7. 提升 `v0.1.1`，再生成 `v0.1.2` 候选版本。
8. 写入错误、记忆和版本记录。
9. 使用包含 `v0.1.2` 的中文 commit 信息提交。
